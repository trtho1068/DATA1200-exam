Accessibility

Here we go through how we used automated tests to try and improve our site, primarily by making it more accessible. We've used Google Lighthouse.

First round

The test was run on November 6th 2023 against this commit.

Our overall accessibility score is 94, not so bad, but there is room for improvement. There is also an accessibility related issue in the SEO section, we will come back to that.

Performance

A few other issues are grouped under performance and relates primarily to images, which at this stage there was only one of.

Setting an explicit width and height is fairly straightforward, when using the <img> tag Intellij will even suggest the appropriate values.

Properly sizing images is trickier. The consequence of too large images is wasted bytes and slower page load. To at least improve on what was, we replace the image on the home page, with two cropped versions. One for viewports > 1200px and one for viewports < 1200px.

To serve images in next-gen formats we convert them to webp. We provide jpg fallbacks for browsers not supporting webp.

Caching resources is not part of DATA1200, so this suggestion we ignore.
